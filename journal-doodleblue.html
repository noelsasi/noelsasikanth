Full story

Two years. Five projects. Multiple clients across three different domains — fashion, fintech, and enterprise HR. This is where I stopped learning React and started really knowing it. The difference is hard to describe until you've crossed it: you stop thinking about how to do things in React and start thinking about what you're building.

The Fashion Platform

The first major project was an e-commerce platform for a Dubai-based fashion client — clothing rental and outright sales, with merchant dashboards, inventory management, customer storefronts, and analytics. Full stack: React on the frontend, Node.js backend, AWS Lambda for serverless microservices. It went live and reached a 25% increase in order completion rate compared to their previous platform.

This was also my first serious TypeScript project. TypeScript at first feels like it slows you down — you're writing more, thinking about types you never thought about in JavaScript. But once you're in a codebase of any size with multiple engineers, TypeScript is the difference between refactoring with confidence and refactoring with fear. I never want to work on a large JavaScript codebase without it again.

SHL: Interview Scheduling at Scale

The second major project was an interview self-scheduling portal for SHL — an enterprise HR and talent assessment company. One portal serving three different user types: recruiters, panellists, and candidates. Each had different permissions and views of the same scheduling data.

The complexity was in the panel configurations: one-to-one, one-to-many, many-to-many interviews — each requiring different calendar logic and invite automation. GraphQL was the right call here, letting the frontend query exactly the data shape it needed for each view without overloading the API with generic endpoints.

NeoGrowth: Fintech Loan Journey

NeoGrowth's NeoCash Insta Web was a mobile-first instant loan application journey. The UX had to be frictionless: PAN validation, Aadhaar verification via DIGIO, eligibility checks, and disbursement — all in a flow that felt like a consumer app, not a banking form. UTM campaign tracking and Firebase Analytics for measuring where users dropped off.

Fintech taught me to think about error states differently. Every step in a financial journey has failure modes that matter: a failed Aadhaar OTP, a PAN that doesn't match, a network drop mid-verification. The user can't just see a generic error — they need to know exactly what happened and what to do next. Clear, precise error handling isn't a nice-to-have in fintech. It's the product.

React Native: Mobile Is a Different World

The last significant chapter at Doodleblue was building two React Native apps for NeoGrowth: a SalesApp for field sales agents and a Customer App for loan management. Both shipped to App Store and Play Store.

Mobile development is genuinely different from web. No browser as a safety net. Platform-specific behaviour that you have to test on real devices. App lifecycle events — backgrounding, foregrounding, push notifications — that don't exist on the web. Camera and secure storage integrations that require native modules. The SalesApp had KYC photo capture, form capture for loan applications, lead assignment, and offline capability in areas with poor connectivity.

Shipping to the App Store for the first time involves a whole category of work that doesn't exist in web: certificates, provisioning profiles, review guidelines, binary uploads. Getting rejected by App Review once — because of a missing privacy permission string — is the fastest way to learn to read the guidelines carefully.

Backend and Infrastructure

The fashion platform and the NeoCash Insta journey both had significant backend components. I built and deployed serverless microservices on AWS Lambda — on-demand compute that scaled with usage and eliminated the cost of always-on servers. Migrated the Insta Journey platform to AWS Amplify from its legacy deployment, cutting deployment time by 50% and backend operational costs by 50%.

The merchant dashboards needed real-time insights: inventory levels, order volumes, payment statuses, stock-out predictions. Building that required Node.js APIs that aggregated data cleanly and served it fast enough for dashboards refreshing on demand. These weren't just frontend projects — they were full-stack systems where the API contract and database design mattered as much as the UI.

Working in a Multi-Client Studio

Two years in a product studio means you see a lot of codebases, a lot of client expectations, and a lot of different ways teams can succeed or fail. I learned Agile done properly: sprint planning that actually sets realistic scope, standups that solve blockers instead of just reporting status, retrospectives where the team is honest.

More than any specific technology, Doodleblue gave me range. I know what it means to pick up a new domain quickly, understand a client's existing system, and deliver production-ready code without months of ramp-up. That's a skill that compounds.
