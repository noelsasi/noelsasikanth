RoamHome

Stepping Into React

Jan 2020 — Jul 2020 · Goa, India

Coming from server-rendered Pug templates at Planys, React felt like a different planet. No full page reloads. State that lives in components. Re-renders that happen automatically when data changes. The mental model shift takes a while — and the best way to make it stick is to build something real with real users.

RoamHome was an early-stage holiday villa rental platform, India-focused, similar to Airbnb in scope. Three distinct surfaces: a guest-facing booking app, a partner dashboard for property owners, and payment flows via Razorpay. I built all three.

The Booking App

The guest side was standard enough: property search with filters, listing pages, availability checks, a booking funnel, Razorpay payment integration, user profiles, and post-stay ratings. Good practice for state management across multiple steps. Redux for the first time — and immediately realising that global state has to be structured carefully or it becomes noise you can't read.

I used Ant Design for UI components, React Router for navigation, Google Maps for property locations, and Axios for API calls. The guest app was mobile-first by necessity — most bookings happen on phones.

The OTA Calendar Problem

The partner dashboard was where the interesting engineering happened. Property owners on RoamHome didn't list exclusively here — they were also on MakeMyTrip, Airbnb, Booking.com, and other Online Travel Agencies simultaneously. Bookings from all those platforms needed to show up in one calendar.

This is a genuinely hard UI problem: you have bookings with different sources, statuses, overlapping date ranges, colour coding per OTA, and a partner who needs to quickly see what's blocked, what's pending, and what's available — all in one view, without switching tabs.

I built a custom calendar component from scratch. No library — none of the off-the-shelf calendar components handled multi-source booking data at this level of density. It had to consolidate ranges from different OTAs, handle partial-day overlaps, render availability clearly, and update in near-real-time as new bookings came in. This reduced booking management time for property owners by 40%.

That component taught me something important: sometimes the right answer is to build it yourself. Not because it's more impressive, but because the problem is specific enough that a general library will either not fit or require more customisation than writing from scratch.

Short Engagement, Real Lessons

RoamHome was a short engagement — January to July 2020. The pandemic disrupted travel and the business with it. But the engineering work was real, and the problems were the kind that don't appear in tutorials: multi-source data synchronisation, property owner workflows, payment edge cases.

It solidified React for me. Not at an expert level yet — but enough to stop fighting the framework and start working with it. That's the inflection point that matters.

Stack: React · Redux · Ant Design · React Router · Razorpay · Google Maps · Axios · SCSS
