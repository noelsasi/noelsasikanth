HiLabs

Everything Comes Together

Feb 2023 — Present · Bengaluru, India

HiLabs solves a specific and unglamorous problem: clinical data accuracy for large US health insurers. Provider directories with wrong phone numbers. Contracts with incorrect reimbursement attributes. Credentialing records that haven't been verified in years. In American healthcare, bad data is a compliance risk, a patient safety issue, and an enormous operational cost. HiLabs' AI stack finds it, flags it, and helps fix it.

I came in as a Senior Software Engineer II. I left that role (twice, via promotion) and am now a Technical Lead. The work has compounded — each system I built became the foundation for something larger.

IQT: From Prototype to Production

IQT — the Intelligent Query Tool — started as an internal demo: could we build a ChatGPT-style interface on top of contract PDFs? Could a user ask "what are the reimbursement rates for this TIN group?" and get an answer with a highlighted reference in the original document?

The demo worked. Then it became a project. Then it went to Elevance — one of the largest health insurers in the United States — and it's been in production since.

The architecture uses a LangChain + RAG pipeline: documents are chunked, embedded, and stored in a vector index. User queries are embedded at runtime and matched to relevant chunks. Those chunks are passed to the LLM as context, which then generates a grounded, cited answer. The frontend — built in React with Material UI — shows the answer alongside the relevant PDF page with highlights and navigation. Document review time dropped by 40%.

I built IQT full-stack: the React frontend, the Node.js backend, and the LangChain pipeline. I also built the Flask PDF Highlighter API — a separate Python service that takes a PDF and a list of strings and returns the annotated PDF with highlights at the exact coordinates of each match. That API is now used across multiple products in the platform.

Pricing Configuration

The Pricing Configuration module shows the full lifecycle of contract ingestion: which files have been processed, what attributes were extracted, what chunks carry evidence for each attribute, and what the processing status is at every step. It's the audit trail for the AI's work — giving compliance teams confidence that the system can be trusted.

This is the kind of product that doesn't look impressive in a demo but is genuinely essential in production. Enterprise customers need to see that the AI isn't a black box. Pricing Configuration is how HiLabs shows its work.

Performance: Intersection Observer at Scale

Provider 360 is a screen that displays complex clinical provider data — accordions with hundreds of nested entries, each expandable, each making API calls. At the scale of Elevance's dataset, rendering everything naively caused visible lag and excessive API load.

I rewrote the scroll behaviour using the Intersection Observer API — both at the top-level page scroll and at the accordion level. Elements are only loaded and rendered as they enter the viewport. Rendering time dropped by 45%. This is now the pattern used across all infinite-scrolling views in the platform.

Micro-Frontends

The HiLabs frontend platform grew from one product to several, each with their own release cadences and teams. A monolithic frontend would mean every team's changes could affect every other team's release.

I contributed to the migration to micro-frontends using Vite Module Federation. Each module (IQT, Pricing Configuration, Provider 360) can now be built and deployed independently. No full-app scans before release. No waiting for other teams to merge. The architecture is still evolving, but the reduction in deployment risk is already visible.

The Unexpected Side Project

One of the more unusual projects was a PWA food ordering platform built for an internal HiLabs use case. The office ordering process was running on WhatsApp — a group chat where people typed orders, someone compiled them, and confusion was frequent.

The replacement was a React PWA with a Node.js backend, AWS Lambda for serverless processing, and a MySQL database. Orders are submitted through the app, compiled automatically, and routed to vendors. Operational dependency on the WhatsApp process dropped by 90%. It's a small system, but it runs in production every day, and that's what matters.

Growing Into Leadership

Two promotions in two years isn't something you plan — it's a byproduct of taking ownership seriously. I went from implementing features to designing them. From following architecture decisions to making them. From writing code to reviewing it and mentoring the engineers writing it.

The hardest shift wasn't technical. It was learning to define OKRs for a team, run sprint planning that actually matches capacity to scope, and communicate tradeoffs to stakeholders who don't care about the implementation details.

The biggest thing I've learned at HiLabs: how quickly a POC becomes a production system that customers depend on. Build like it's already in production. Because it will be, sooner than you think.
